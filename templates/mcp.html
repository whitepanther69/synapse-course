Behind the tutor

How SYNAPSE uses MCP and multiple AI models

You don't need to know any of this to use SYNAPSE. But if you're curious about why it explains things the way it does — patiently, visually, one clear step at a time — here's what's happening under the hood, in plain language.

The big idea

What makes SYNAPSE different?

Most coding tutors hand you a single chatbot and hope for the best. SYNAPSE was built differently. It was designed first for people who find ordinary learning tools exhausting — learners with ADHD, dyslexia, or anyone whose attention is the thing in short supply — and built in a way that benefits everyone.

In practice that means explanations that come with a picture, not a wall of text. Hints that arrive in stages instead of dumping the answer on you. A tutor that gently checks in when you've gone quiet, and never makes you feel slow. None of that is one clever trick — it's a lot of careful choices about how help should feel. (There's more on that on the About page.)

One of those choices is what this page is about: SYNAPSE doesn't lean on a single AI. It quietly uses several, each doing the part it's best at.

The why

Why more than one AI model?

Different AI models are good at different things — a bit like asking one friend to explain an idea, another to sketch it out, and a third to help you plan. SYNAPSE does the same, automatically:

🗣️

One model explains

The patient, step-by-step teaching, and the back-and-forth when you're stuck.

🛠️

One brings ideas to life

Worked examples and visual guides that let you see an idea, not just read about it.

🧭

One adds planning & structure

Shaping a clear answer before it's written, so the explanation actually flows.

You never have to pick or manage any of this; it all happens behind the scenes. You just ask your question. And because more than one model is involved, if one is ever unavailable an automatic backup quietly takes over and the tutor keeps working. What you actually feel is the only part that matters: help that's steadier and clearer, without needing to think about which model is doing what.

The how

What is MCP?

The piece that makes this work cleanly is the Model Context Protocol (MCP). Originally introduced by Anthropic, it's an open standard for connecting AI models to tools.

Instead of one AI improvising whatever it happens to produce, SYNAPSE gives it a clear set of purpose-built tools — explain this code, check it for security problems, draw a diagram, generate practice — and MCP is the shared language that lets the models use those tools the same dependable way every time.

What it means for you It comes down to one thing: consistency. You get the same reliable kinds of help every session, instead of hoping for a good day. It's a deliberate, structured pipeline — not a model deciding its own routing on the fly.
The tools

The seven teaching tools

SYNAPSE runs its own MCP server offering seven teaching tools — seven distinct ways of helping you understand and improve code:

📖

Explain Code

Plain-language explanation of what a piece of code does, and the ideas behind it.

🔒

Security Check

Reviews your code for vulnerabilities, mapped to real-world weaknesses (CWE) and the OWASP Top 10.

📊

Visual Guide

Turns an idea into a diagram or visual walkthrough so you can see the structure, not just read it.

🔍

Analyze Code

A fuller look at the quality and structure of what you've written.

🔧

Suggest Improvements

Concrete, friendly suggestions to make your code better.

💪

Practice Exercises

Targeted exercises, with hints that arrive in stages.

⚡

Run & Test

Actually runs and tests your code, so the feedback is real, not guesswork.

Open & reproducible

Open and reproducible

SYNAPSE was built as an Honours research project, and it's openly documented. The code is on GitHub under the MIT licence, and a citable software archive is preserved on Zenodo with a permanent DOI.
